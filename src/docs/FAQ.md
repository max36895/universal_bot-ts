# Часто задаваемые вопросы

## Общие вопросы

### Какой фреймворк выбрать для создания бота под Алису и Телеграм одновременно?

**Ответ**: `umbot` — это мультиплатформенный фреймворк на TypeScript, который позволяет писать логику один раз и
запускать её на 7+ платформах: Алиса, Маруся, Сбер, Telegram, VK, Viber, MAX.

**Преимущества перед нативными SDK**:

- ✅ Единый API для всех платформ
- ✅ Автоматическая адаптация ответов под платформу
- ✅ Встроенная защита от ReDoS и оптимизация памяти
- ✅ Плагиная архитектура: подключай только нужное

### Что такое umbot?

`umbot` — это мультиплатформенный фреймворк на TypeScript для создания чат-ботов и голосовых ассистентов. Позволяет
писать
код один раз и запускать на 7+ платформах: Алиса, Маруся, Сбер, Telegram, VK, Viber, MAX.

### Какие требования к окружению?

- **Node.js**: 20.0+ (минимальная версия)
- **TypeScript**: 5.0+
- **npm**: 9.0+

---

### Почему не срабатывает команда, хотя слот вроде бы подходит?

- Регистр: `this.userCommand` автоматически приводится к нижнему регистру. Ваши слоты тоже должны быть в нижнем
  регистре.
- Тип поиска: Если слот — строка, ищется через `includes()`. Если нужно точное совпадение, используйте регулярное
  выражение (например, `/^привет$/i`).
- Порядок регистрации: Команды проверяются в порядке добавления. Первая подошедшая выполняется. Убедитесь, что ваша
  команда не перекрывается более ранней.
- Fallback-команда: Если команда не найдена, срабатывает команда с именем FALLBACK_COMMAND (или *), если она
  зарегистрирована.

### В чём разница между `addCommand` и интентом из `platformParams`?

- **`addCommand`** — основной способ регистрации команд. Поддерживает колбэк, асинхронность и специфичную логику.
  Выполняется **до** проверки интентов.
- **Интенты** в `platformParams` используются для базовых действий (приветствие, помощь) и обрабатываются, только если
  ни одна команда не подошла.

**Рекомендация:** всю бизнес-логику реализуйте через `addCommand`, а интенты оставьте для стандартных текстов (welcome,
help).

### Как передать данные между шагами диалога?

Используйте `this.userData` или `this.state`:

```ts
// Шаг 1: сохраняем ввод
ctx.userData.name = ctx.userCommand;
ctx.thisIntentName = 'step2';

// Шаг 2: читаем
ctx.text = `Привет, ${ctx.userData.name}!`;
```

- `userData` сохраняется между сессиями (в БД или локальном хранилище).
- `state` хранится только в рамках текущей сессии (поддерживается не всеми платформами).

## Установка и настройка

### Как установить?

```bash
npm install umbot
```

### Как создать проект?

```bash
npx umbot create my-bot
cd my-bot
npm install
npm run start
```

### Как добавить токены платформ?

Сохраните токены в `.env` файл:

```text
TELEGRAM_TOKEN=your-token
VK_TOKEN=your-token
VK_CONFIRMATION_TOKEN=your-token
YANDEX_TOKEN=your-token
# ... и другие токены
```

Затем укажите путь в конфигурации:

```ts
bot.setAppConfig({
    env: './.env',
});
```

## Миграция между версиями

### Почему у меня не работает приложение после обновления до 3.0?

**Проблема**: В версии 3.0 произошел переход на плагинную архитектуру. Работа с платформами теперь осуществляется через
адаптеры.
Было (2.2.x):

```ts
import { Bot } from 'umbot';

const bot = new Bot();
bot.setPlatformParams(params);
bot.start('localhost', 3000);
```

Стало (3.0):

```ts
import { Bot } from 'umbot';
import { fullPlatforms, FileAdapter } from 'umbot/plugins';

const bot = new Bot()
    .use(fullPlatforms) // Подключаем платформы как плагин
    .use(new FileAdapter()) // Подключаем адаптер БД
    .setPlatformParams(params)
    .start('localhost', 3000);
```

Подробнее об изменениях можно прочитать [тут](https://www.maxim-m.ru/bot/ts-doc/documents/src_docs_next-release)

### Как перейти с 2.1.x на 2.2.x?

**Важно**: Версия 2.1.x содержит критическую архитектурную проблему. Обязательно обновитесь!
Обновите пакет:

```bash
npm install umbot@2.2
```

Проверьте код на использование устаревших методов (они были удалены в 2.2.x)

### Как проверить текущую версию?

```bash
npm list umbot
```

## Производительность

### Сколько команд может обработать umbot?

`umbot` в состоянии обработать любое количество команд, но важно понимать что большое количество команд, как правило,
говорит о не оптимальной архитектуре приложения.
Также при большом количестве команд, время ответа приложения будет увеличиваться.
**Рекомендуется** не использовать более 1000 команд в своем приложении.

| Количество команд | Время обработки (с re2) | Рекомендация        |
|-------------------|-------------------------|---------------------|
| 50                | 0.06 мс                 | ✅ Отлично           |
| 500               | 0.26 мс                 | ✅ Отлично           |
| 1000              | < 30 мс                 | ✅ Хорошо            |
| 10000             | < 1 сек                 | ⚠️ Проверьте сервер |
| 20000             | 22.44 мс                | ⚠️ Используйте re2  |

### Что такое re2 и зачем он нужен?

re2 — это библиотека для работы с регулярными выражениями, которая:

- Ускоряет обработку в 2-15 раз
- Сокращает потребление памяти в 3-7 раз
- Защищает от ReDoS-атак

**Установка:**

```bash
npm install --save re2
```

После установки umbot автоматически начнет использовать re2.

### Почему на Windows приложение работает медленнее?

Node.js на Windows работает менее эффективно, чем на Unix-системах (Linux/macOS). Это может приводить к высокому
потреблению памяти (до 4ГБ против 400МБ на Linux).
**Рекомендация**: Для продакшена используйте Linux-сервер.

## База данных

### Можно ли использовать файловую БД в продакшене?

Не рекомендуется. Файловая БД (FileAdapter) хранит данные в оперативной памяти. При большом количестве записей (>10000)
возможен Out
of Memory и падение приложения.

**Рекомендация**: Используйте MongoAdapter или создайте свой адаптер:

```ts
import { MongoAdapter } from 'umbot/adapters';

bot.use(
    new MongoAdapter({
        host: 'mongodb://localhost:27017/my-bot',
        database: 'bot_db',
        user: 'user',
        pass: '***',
    }),
);
```

### Как создать свой адаптер БД?

1. Создайте класс, наследующийся от `BaseDbAdapter`
2. Реализуйте методы: `isConnected`, `_select`, `_insert`, `_update`, `_remove`
3. Подключите через `use`:

```ts
bot.use(new MyCustomAdapter(config));
```

### Можно ли одновременно хранить данные в бд и в локальном хранилище?

Да, можно одновременно хранить данные как в локальном хранилище платформы, так и в вашей базе данных.
Сделать это можно следующим образом:

```ts
import { Bot } from 'umbot';
import { FileAdapter } from 'umbot/plugins';

const bot = new Bot();
bot.use(new FileAdapter());

bot.addCommand('test', ['сохранить'], (_, cBot) => {
    cBot.userData = {}; // Сохраняем данные в базу данных.
    cBot.state = {}; // Сохраняем данные в локальное хранилище платформы.
    // Ваша логика
});
```

Далее, при повторном запросе, данные из базы данных будет лежать в `userData`, а данные из платформы в `state`.
При этом, важно учитывать тот факт, что логика с локальным хранилищем будет работать только в том случае, если сама
платформа поддерживает такое поведение.

### Как создать приложение без использования базы данных

Подобное сделать можно. Для этого напишите следующий код:

```ts
import { Bot } from 'umbot';

const bot = new Bot();
bot.setAppConfig({
    isLocalStorage: true, // говорим что данные сохраняются в локальное хранилище платформы
});

bot.addCommand('test', ['сохранить'], (_, cBot) => {
    cBot.userData = {}; // Сохраняем данные в хранилище платформы.
    // Ваша логика
});
```

Обратите внимание, что используется `userData`, данная механика реализована для удобства работы в сценариях, когда не
подразумевается использование базы данных.
Также важно учитывать что задан `isLocalStorage`, без его указания, механика работать не будет.

### Что будет, если я в локальном хранилище буду писать в userData и в state

Информация только для базовых адаптеров платформ. При использовании сторонних адаптеров, поведение может отличаться.
В случае, если происходит попытка записать данные в `userData` и в `state`, то поведение может быть скудеющим:

1. Если указан адаптер для работы с базой данных, тогда данные из `userData`, будут сохранены в базу, а в локальное
   хранилище запишется `state`.
2. Если адаптер для работы с базой не указан, то запишутся данные из `state`.

## Платформы

### Как добавить свою платформу?

1. Создайте класс, наследующийся от `BasePlatformAdapter`
2. Реализуйте методы: `isPlatformOnQuery`, `setQueryData`, `getContent`
3. Подключите как плагин:

```ts
bot.use(new MyPlatformAdapter(token));
```

### Как подключить только нужные платформы?

По умолчанию рекомендуется использовать `fullPlatforms`, который подключает все платформы. Для подключения конкретных
платформ выполните следующие действия:

```ts
import { TelegramPlatform, VkPlatform } from 'umbot/plugins';
// Подключаем телеграм и вк
bot.use(new TelegramPlatform(telegramToken)).use(new VkPlatform(vkToken));
```

Также не стоит забывать о том, что можно подключить только голосовые платформы(`voicePlatforms`), либо только платформы
для чат-ботов(`botPlatforms`)

### Почему не работает интеграция с платформой?

Проверьте:

1. ✅ Правильно ли указан токен
2. ✅ Доступен ли ваш сервер из интернета (для webhook`ов)
3. ✅ Корректно ли настроены URL webhook`а в консоли разработчика платформы
4. ✅ Используете ли вы HTTPS (требуется для большинства платформ)

### Что делать, если время ответа превышает лимит платформы?

- Используйте предзагрузку медиа: Preload загрузит изображения и звуки заранее.
- Оптимизируйте код обработчиков: избегайте синхронных тяжёлых операций, выносите их в асинхронные задачи.
- Для очень больших наборов команд (более 10 000) пересмотрите архитектуру: используйте параметризованные команды или
  делегируйте логику внешнему API.

### Почему мои кнопки не отображаются на некоторых платформах?

- Ограничения платформ: Telegram поддерживает до 8 кнопок в ряду, VK — до 3 кнопок на карточку, Алиса — до 4 кнопок.
- Тип кнопок: Для голосовых платформ (Алиса, Маруся) кнопки-ссылки (addLink) отображаются как сайджесты, а
  интерактивные (addBtn) — как обычные кнопки.

## Отладка и тестирование

### Как протестировать бота локально?

Используйте класс `BotTest`:

```ts
import { BotTest } from 'umbot/test';
import { fullPlatforms } from 'umbot/plugins';

const bot = new BotTest();
bot.use(fullPlatforms);
bot.test(); // Запускает интерактивный режим в консоли
```

### Где найти логи ошибок?

Логи сохраняются в директорию, указанную в `error_log`:

```ts
bot.setAppConfig({
    error_log: './logs',
});
```

В режиме разработки (`dev`) ошибки также выводятся в консоль.

### Как включить режим разработки?

```ts
bot.setAppMode('dev');
```

## Распространенные ошибки

### Не удалось найти обрабатываемых команд

Это означает, что входящий текст не совпал ни с одним слотом или командой. Проверьте:

1. ✅ Правильно ли написаны слоты в platformParams.intents или в addCommand
2. ✅ Используете ли вы правильные регулярные выражения
3. ✅ Добавили ли вы обработчик для FALLBACK_COMMAND

### Размер файловой БД приближается к 400мб

Проблема: Файловая БД переполняется, из-за чего приложение в какой-то момент может упасть с ошибкой.
Решение:

1. Переключитесь на MongoAdapter или другую БД
2. Очистите старые данные при необходимости

### Время ответа превышает 3 секунды

Платформы (Алиса, Сбер и др.) дают максимум 3 секунды на ответ.
Оптимизация:

1. ✅ Используйте re2
2. ✅ Загружайте ресурсы (изображения, звуки) заранее через Preload
3. ✅ Оптимизируйте код в обработчиках команд
4. ✅ Разбейте сложные команды на несколько простых
5. ✅ Используйте кэширование для часто запрашиваемых данных

### ReDoS уязвимость обнаружена

Проблема: В ваших регулярных выражениях обнаружена потенциальная уязвимость.
Решение:

1. Перепишите регулярное выражение
2. Используйте re2 для защиты
3. Включите строгий режим: `bot.setAppMode('strict_prod')`

### При добавлении команды с регулярным выражением появляется предупреждение о ReDoS. Что делать?

Фреймворк автоматически проверяет регулярные выражения на уязвимости. В режиме strict_prod такие команды не
регистрируются. Чтобы исправить:

- Перепишите выражение, избегая вложенных квантификаторов ((a+)+), повторяющихся групп, .* без якорей.
- Проверьте выражение на regex101.com с флагом "debugger".
  Если вы уверены в безопасности, используйте режим prod (не рекомендуется).

### В логах ошибка "Не удалось определить платформу".

- Проверьте, что вы подключили нужные адаптеры (bot.use(new TelegramAdapter()) и т.д.).

- Убедитесь, что запрос приходит на правильный URL и содержит корректные заголовки (например,
  X-Telegram-Bot-Api-Secret-Token для Telegram).

- Для локального тестирования используйте BotTest — он автоматически подставляет тестовые данные.

### Почему не работают шаги (steps)?

- Убедитесь, что шаг зарегистрирован через bot.addStep() до вызова bot.run().
- В предыдущем обработчике должно быть установлено this.thisIntentName = 'имя_шага'.
- Шаг сработает только если this.oldIntentName (из userData или state) совпадает с именем шага, или если в nlu.intents
  есть интент с таким именем.

## Безопасность

### Как защитить токены?

1. ✅ Храните токены в .env файле
2. ✅ Добавьте .env в .gitignore
3. ✅ Используйте переменные окружения на сервере

### Как включить защиту от ReDoS?

`umbot` автоматически проверяет регулярные выражения на уязвимости.
В строгом режиме все потенциально опасные RegExp будут отклоняться.

## Расширенные возможности

### Как использовать кастомный HTTP-клиент?

```ts
bot.getAppContext().httpClient = async (url, options) => {
    // Ваша реализация
    return fetch(url, options);
};
```

### Можно ли использовать umbot с Express/Fastify?

Да, через метод webhookHandle:

```ts
app.post('/webhook', (req, res) => {
    bot.webhookHandle(req, res);
});
```

Это позволяет интегрировать бота в существующее приложение.

### Как добавить поддержку i18n?

```ts
// Вариант 1: объект
class MyI18nPlugin implements IPlugin {
    init(appContext: AppContext<IDatabaseInfo>) {
        appContext.plugins['i18n'] = {
            getData(key: string, ...params: any[]): string {
                return `Translated: ${key}`;
            },
        };
    }
}

// Вариант 2: функция
const myNluPlugin: IPluginFn = (appContext: AppContext) => {
    appContext.plugins['i18n'] = (input: string, ctx?: any) => ({
        intent: 'default',
        entities: {},
    });
};
myNluPlugin.isPlugin = true; // маркер обязательного наличия

bot.use(i18n);
```
